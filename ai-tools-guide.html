// 先給妳一張地圖

先有一張地圖：AI 怎麼分？

分辨任何一個 AI 工具，其實只要問兩個問題：它長什麼樣子？（介面）以及 它會不會自己動手做事？（能力）。看懂這兩個，妳就不會再被一堆名詞嚇到。

① 依「介面形式」分 —— 它長什麼樣子

類型長相代表對新手
純網頁聊天開瀏覽器打字ChatGPT、Claude.ai、Gemini最簡單，但只會「講」
● 桌面式 App（GUI）裝在電腦、滑鼠點Claude／ChatGPT 桌面版甜蜜點：好上手又會動手
● 終端機版（CLI）黑底命令列打指令Claude Code、Codex CLI最強最自動，門檻較高
IDE 外掛／編輯器寫程式的軟體裡Cursor、Windsurf偏工程，我們先不必碰
● 自動化平台拉流程圖／設觸發Zapier、Make、n8n串「自動跑」的流程

② 依「能力」分（這個最關鍵）

聊天式 AI：一問一答，給妳答案，但不會自己動手。
代理式 AI（Agent）：會自己規劃、用工具、讀寫檔案、執行任務、出錯還會自己修 ——「會動手把一件事做完」。

我們課堂要學的 Claude Code，還有 Codex，都屬於「代理式」。它們特別的地方是 同一個引擎、跨多種介面（終端機／桌面 App／IDE／雲端都能用）。

// 01 ・ 終端機

終端機版 AI

住在命令列（黑底白字視窗）裡的 AI。妳用日常語言下指令，它能讀檔、改檔、執行、自我修正 —— 不只回妳一句話，而是幫妳把事情做完。看起來最專業，門檻也最高。

// 02 ・ 桌面

桌面式 AI（App）

有視窗、用滑鼠點的 AI App。介於「難用的終端機」與「功能較少的純網頁」之間 —— 對我們來說，是好上手又能動手的甜蜜點。這也是我們課堂主要用的。

🔑 2026 對新手最重要的好消息

Claude 桌面版的「Code」分頁，現在 不需要打終端機 就能做那些「會動手」的事 —— 有視覺化的檔案對照、內建預覽與編輯器，每個動作都逐次問過妳，官方還特別建議「新手用 Ask 權限模式」。對「看到黑底命令列就怕」的妳，這是超溫柔的入口。

// 03 ・ 自動化

自動化 AI 與 Agent（代理）

能自己規劃、用工具、跨步驟把任務「自動跑完」的 AI。核心精神很簡單：把做過兩次以上的事，變成一鍵重播的流程。

自動化 / Agent
平台定位價格
Zapier最易上手免費 100 tasks/月起
Make最划算約 US$9/月（免費 1,000 點）
n8n最彈性、可自架自架免費
Claude Skill／Computer UseAI 原生代理（欣妙現在在用的工作流）含於 Claude 方案
ChatGPT AgentAI 原生代理含於 ChatGPT 方案

⚠️ 最大風險：AI 會「很有自信地犯錯，然後自動繼續跑」

所以記得三句話：人在迴路、先小量測試、重要的事先確認。這也是「權限模式」存在的原因 —— 它會在動手前先問妳。

// 04 ・ 我們的主角

Claude Code（我們課堂的主角）

Anthropic 的代理式 AI 工作助手 —— 讀寫檔案、執行指令、做網頁、串工具。同一個引擎，跨終端機／桌面 App／IDE／網頁四種介面。我們就是要用它，幫妳把點子做出來。

Claude Code
項目內容
怎麼用（我們課堂）桌面版 App 的 Code 分頁，滑鼠點、不碰終端機
方案免費版不能用；Pro $20/月就很夠用（Max 是重度用戶才需要）
之後會認識的概念CLAUDE.md（給 AI 的說明書）・權限模式・Skill（可重複的配方）—— 現在聽過就好，之後一堂堂帶
新手最溫柔的入口桌面 App ＋ Ask 權限模式（動手前先問妳）
  • 為什麼我們選它：對非工程的人最友善、繁體中文好、能串 Google 工具 —— 而且欣妙自己就是從零學起、一路做出網頁和表單的活例子，所以我知道怎麼陪妳。
  • 誠實說一句：Codex 也做得到類似的事。我們選 Claude Code，是因為它最友善、我最熟、最能一路陪妳走。
// 05 ・ 也認識一下

OpenAI Codex（順便認識）

2026 的「Codex」是 OpenAI 全新的代理式工具（跟 2021 那個同名舊模型無關）。一個帳號跨多形態：終端機、IDE、雲端、ChatGPT 內、GitHub。妳不用學它，認識一下就好，這樣別人提到妳不會慌。

// 06 ・ 妳能怎麼用

給陪伴工作者的 AI 應用

我們這群人 —— 做陪產、哺乳母嬰、諮商、療癒身心靈、教育、陪伴、女人圈的女性 —— 有個共同點：會接觸很私密的個案資料。所以「怎麼用」和「哪裡不能用」一樣重要。

🚫 最重要的紅線：個案隱私

不要把可識別的個案資料（姓名、生日、聯絡方式、可辨識細節）貼進一般 AI —— 多數主流 AI 並不是為醫療／諮商情境設計的，等於把資料送進公共領域。

  • 要用就先去識別化（用代號、抽掉可辨識細節）—— 欣妙的陪產逐字稿就是這樣處理的
  • 知情同意：讓個案知道妳用 AI 做什麼、她可以拒絕
  • 人工核對：AI 會「自信地寫錯」，紀錄一定要自己看過
  • AI 是助理，不是專業者，不能取代妳的判斷與陪伴

來源：NBCC、ACA 美國諮商學會、APA、NPR

// 07 ・ 指令小抄

好用的指令範本庫

把【】換成妳的內容，貼給 AI 就好。⚠️ 沒去識別化的個案資料，不要放進一般 AI。

✍️ 內容創作
你是溫暖但有骨的女性身心靈文案夥伴。請把下面這個感觸，寫成一篇 IG 貼文（300–500 字），語氣溫柔真誠、不浮誇、結尾不要硬銷售。主題／感觸：【…】 對象：25–45 歲的媽媽
請用這個結構寫貼文：1) 抓一個客戶私下擔心、檯面上沒人說的痛點 2) 帶入一個有研究依據的事實（年份/數字＋來源）3) 點出市面上大家通常怎麼說 4) 用「身心靈整合」視角給我的結論。主題：【…】
同一個主題給我三版貼文：V1 深度孕產身心靈(300–800字)｜V2 生活輕鬆版(200–500字)｜V3 短金句版(150字內)。主題：【…】
幫我為這篇內容想 10 個吸引人的開頭第一句（hook），溫暖、能讓媽媽停下來。內容：【…】
📩 行政與溝通
幫我寫一封溫暖的信給報名學員，說明【上課時間／匯款方式／課前準備】，親切清楚不官腔。
我需要婉轉地【婉拒／改期】，對象【…】，情況【…】。幫我寫一則誠懇、不給對方壓力的訊息。
我要辦【活動】，幫我列出報名表單需要的欄位，並為每題寫好溫暖清楚的說明。
📚 備課與教學
我要教【主題】，對象【…】，時間【60】分鐘。幫我設計流程：開場、核心觀念、實作練習、收尾，附 3 個討論問題與 1 個回家小任務。
請把【主題或理論】用白話解釋給完全不懂的媽媽聽，舉生活化例子，並標出哪些有研究依據、哪些是觀點。
🔎 研究與查證（第 2 堂會深入）
請研究【主題】。規則：優先學術(PubMed/Cochrane)與官方(WHO)，交叉比對至少兩種類型；每個主張附來源連結；英文用中文摘要；推測處標註「推測」。
有人說「【某說法】」，有研究依據嗎？請查證共識、相反證據與來源連結。不確定就說不確定。
🗂 整理與摘要（先去識別化）
這是已「去掉姓名、改成代號」的筆記，幫我整理成 5 點重點＋給對方 1 句溫暖回饋：【…】
這段逐字稿（已去識別化），幫我抓出：3 個金句、2 個情緒轉折、1 個可延伸成貼文的題目。【…】
這是我口述的心得逐字稿，幫我整理成通順、保留我口吻的文章，不要過度修飾：【…】
🌱 思考與成長夥伴
我卡在【…】，請先不要給答案，先問我 5 個好問題，幫我想清楚。
我提供【服務】，客群【…】。幫我腦力激盪 3 種定位角度與對應定價邏輯，各自優缺點。
🛠 Claude Code 專用（操作型）
幫我做一頁【服務／活動】網頁。風格：溫暖療癒、玫瑰粉、留白多、手機也好看。內容：【簡介／方案／報名】。做好給我預覽。
幫我用 Google 表單做一份【活動】報名表，欄位包含【…】，並把回應連到試算表方便管理。
我每次都會做這件事：【描述流程】。幫我打包成一個可重複呼叫的 Skill，下次一句話跑完。
// 帶妳記住的重點

看完這麼多，其實只要記住這些

一句話說我們在學什麼

「大部分 AI 只會跟妳聊天、給答案；我們要學的 Claude Code，是會自己動手把東西做出來的 AI —— 幫妳生網頁、做表單、整理資料。」

我們的課會怎麼走

1 ・ 純終端機最正統、最有成就感，但第一關容易勸退。我們先不走這條。
2 ・ 先桌面 App → 再看情況 我們走這條前段用桌面 App 快速有成果、建立信心；之後有需要，再溫柔帶進終端機。
3 ・ 全程桌面 App最無痛，但少了終端機的彈性與成就感。

幾句想送給妳的話

  • 聊天 AI 是顧問，代理 AI 是員工 —— 一個給建議，一個幫妳做完。
  • 把做過兩次以上的事，變成一鍵重播的流程。
  • AI 會很有自信地犯錯 —— 所以「人在迴路、先小量測試、重要的事先確認」。
  • 妳不需要變成工程師，妳只需要願意開口、願意試。

裡面每個事實都查證過官方來源，推測之處都有標。
